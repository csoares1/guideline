Para iniciar nossos estudos devemos levar em conta algumas premissas básicas que iremos seguir para traçar a nossa padronização.

  1. Quanto menor é nossa padronização, mais confuso fica o trabalho, tornando também difícil a substituição de recursos envolvidos em projetos. Eliminar estes problemas é o que motiva a padronização.
  2. Quanto maior é nossa padronização, mais portáteis ficam os sistemas prontos e mais fácil fica a troca de profissionais. No entanto, quanto mais padronizado mais difícil fica para gerenciar e fazer com que as normas sejam cumpridas ou mesmo lidas pelos usuários das mesmas. Além do mais, muita amarração pode diminuir a performance e gerar retrabalho, pois é mais fácil de encontrar padrões descumpridos.
  3. Como regra geral, podemos assumir que a MELHOR padronização é aquela que É SEGUIDA. Sendo assim, esta não deve ser capaz de gerar ambigüidades e casos de arbitragem, não pode ser muito extensa e deve ser amplamente divulgada.
  4. Por se tratar de um documento novo para o projeto, este possivelmente receberá alterações para melhor se adaptar. Assim, é importante que todos verifiquem essas atualizações.
  5. É fato que hoje o projeto não segue padrão bem definido, é de extrema importância o empenho de todos para que os novos projetos sigam os padrões descritos neste, e que à medida que os projetos já existentes sofram alguma alteração, que eles sejam também padronizados.

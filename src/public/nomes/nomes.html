Nomes Significativos

  1. Use nomes que revelem seu próposito. Pois, bons nomes levam tempo, mas economiza mais. Todos que lerem seu código (incluindo você) ficarão agradecidos

    O nome de função, variável ou classe deve responder a todas as questões. Ele deve dizer porque existe, o que faz e como é usado. Se o nome requer comentário ele não revela seu propósito

    int d; // tempo decorrido de dias

    O nome d não revela nada

    int elapseTimeInDays

    elapseTimeInDays indica seu propósito

  2. Nomes devem ser pronunciáveis

  3. Use nomes passíveis de busca

    Imagine uma variável que seja "e" o que não é aconselhável agora imagine ter que busca-la. É uma letra muito comum, devido isso nomes longos se sobressaem dos curtos.

  1. Faça distintinções significativas.

    Palavras muito comuns são redundantes. O nome de uma variável jamais deve conter a palvra "variável"

    Se os nomes precisam ser distintos, então devem ter significados distintos.

  1. Evite Informações Erradas.

    Por exemplo não se refira a um grupo de contas como accountList a menos que realmente seja uma lista
